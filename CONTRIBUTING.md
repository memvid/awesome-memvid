# Contributing to Awesome memvid

Thank you for your interest in contributing to **Awesome memvid**!  
This project is a curated list of high-quality resources related to **memvid**, and we welcome contributions from the community.

Please take a moment to review the guidelines below before submitting a pull request.

---

## What You Can Contribute

You can help by adding:

- Research papers and surveys
- Open-source tools and libraries
- Datasets and benchmarks
- Tutorials, blog posts, and talks
- Real-world projects and applications
- Any other high-quality resource relevant to memvid

---

## Contribution Guidelines

To keep this list useful and consistent, please follow these rules:

### General Rules
- **Quality over quantity** — only add valuable and relevant resources
- The resource must be **freely accessible**
- Prefer **GitHub repositories or official project pages**
- Avoid duplicate entries
- No advertisements or self-promotion-only links

### Formatting Rules
- Use the following format for entries:
  ```markdown
  - [Resource Name](link) – Short, clear description.
  ```
- Keep descriptions concise (one line is usually enough)
- Add new entries to the **most appropriate section**
- Keep sections **alphabetically sorted** when possible

### Scope

A resource should be clearly related to at least one of the following:

* Video memory
* Long-video understanding
* Temporal reasoning over video
* Multimodal (video + text + memory) systems
* Video-based agents or applications

If the connection to memvid is unclear, please explain it briefly in the pull request description.

---

## Pull Request Process

1. Fork the repository
2. Create a new branch for your changes
3. Add your resource following the guidelines above
4. Submit a pull request with a **clear title and description**
5. Be responsive to feedback or requested changes

---

## Keeping the List Healthy

You can also contribute by:

* Removing dead or outdated links
* Updating descriptions
* Suggesting better categorization
* Reporting issues or inconsistencies

---

## Code of Conduct

Please be respectful and constructive in all interactions.
This project follows standard open-source community norms.

---

Thank you for helping make **Awesome memvid** better!
